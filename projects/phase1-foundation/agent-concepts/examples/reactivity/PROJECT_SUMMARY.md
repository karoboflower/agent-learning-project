## 🎯 核心功能演示

### 传感器系统
```
✓ 温度传感器 - 监控环境温度（15-40°C）
✓ 资源传感器 - 扫描可用资源（水、食物、能量）
✓ 威胁检��器 - 检测环境威胁（障碍、捕食者、危险）
✓ 机会扫描器 - 发现潜在机会（资源缓存、捷径、安全区）
```

### 响应机制
```
环境变化 → 传感器检测 → 事件生成 → 优先级排序 → 响应执行
    ↓          ↓           ↓          ↓          ↓
  温度35°C  读取数据   CRITICAL事件  队列最前  紧急冷却
```

### 优先级系统
```
CRITICAL (0) → 立即响应（极端温度、严重威胁）
HIGH (1)     → 高优先级���异常环境、一般威胁）
MEDIUM (2)   → 中优先级（资源不足、机会）
LOW (3)      → 低优先级（一般信息）
```

## 🆚 与自主性的对比

| 维度 | 反应性 | 自主性 |
|------|--------|--------|
| **驱动方式** | 外部事件驱动 | 内部目标驱动 |
| **行为特点** | 被动响应 | 主动执行 |
| **决策依据** | 环境变化 | 内部状态和目标 |
| **示例路径** | `examples/reactivity` | `examples/autonomy` |
| **核心代码** | 传感器监控循环 | 任务执行循环 |

## 💡 关键学习点

### 1. 反应性的本质
```
环境 → Agent → 响应
（外部） （感知） （动作）
```

### 2. 实现要点
- ✅ 持续监控环境
- ✅ 快速检测变化
- ✅ 及时做出响应
- ✅ 避免过度反应
- ✅ 按优先级处理

### 3. 常用模式
- **轮询模式**: 定期检查环境状态
- **事件驱动**: 订阅环境变化事件
- **反应式流**: 使用RxJS等库处理数据流
